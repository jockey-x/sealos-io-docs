## Sealos 认证与模板查询 API 文档

本文档整理了 Sealos 第三方接入中常用的认证接口与模板查询接口，包含：

* 账号密码登录，获取 Global Token

* 使用 Global Token 换取 Region Token

* 获取当前用户 kubeconfig

* 轮转 kubeconfig

* 校验 kubeconfig

* 获取应用模板列表

* 获取应用模板详情

## 1.  基础说明

### 1.1 服务域名

* `desktop` API 基础地址：`https://<sealos-domain>`

* `template` API 基础地址：`https://<template-domain>`

* applaunchpad   API 基础地址：`https://<applaunchpad-domain>`

示例：

* `desktop`：`https://192.168.10.70.nip.io`

* `template`：`https://template.192.168.10.70.nip.io`

* `applaunchpad`：`https://applaunchpad.192.168.10.70.nip.io`

### 1.2 鉴权约定

不同接口使用的鉴权方式不同：

* `desktop` 登录态接口使用 Token 作为 `Authorization` 请求头值，直接传 token 字符串，不需要 `Bearer ` 前缀。

* 后续涉及 `kubeconfig` 鉴权的接口，使用 `encodeURIComponent(kubeconfig)` 之后的结果作为 `Authorization` 请求头值。

* 本文档中的模板查询接口是公开查询接口，不需要鉴权。

## 2. 认证流程

第三方接入建议按照以下顺序获取访问凭证：

1. 调用 `desktop /api/auth/password`，获取 Global Token。

2. 调用 `desktop /api/auth/regionToken`，获取 Region Token、App Token 以及 kubeconfig。

3. 后续：

   * 调用 `desktop` 登录态接口时，`Authorization` 传 Region Token 或 App Token。

   * 调用依赖 kubeconfig 的接口时，`Authorization` 传 `encodeURIComponent(kubeconfig)`。

## 3. 获取 Global Token

### 接口

* 方法：`POST`

* 路径：`/api/auth/password`

* 服务：`desktop`

### 请求头

```http
Content-Type: application/json
```

### 请求体

| 字段         | 类型     | 必填 | 说明                      |
| ---------- | ------ | -- | ----------------------- |
| `user`     | string | 是  | 用户名。0326 更新后，该字段没有字符限制。 |
| `password` | string | 是  | 登录密码。                   |

### password 字段规则

0326 更新后，文档约定如下：

1. 长度至少 8 个字符。

2. 不能全是空白字符，必须至少包含一个非空白字符。

说明：

* 用户未注册时，服务端支持自动注册用户，并初始化可用区信息。

### 请求示例

```bash
curl -s -X POST 'https://<sealos-domain>/api/auth/password' \
  -H 'Content-Type: application/json' \
  -d '{
    "user": "admin",
    "password": "8f4c75cexxxxxxxxxxxxxxxxefa675df"
  }'
```

### 成功响应

```json
{
  "code": 200,
  "message": "Successfully",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.xxx",
    "needInit": false
  }
}
```

### 响应字段说明

| 字段              | 类型      | 说明                                                 |
| --------------- | ------- | -------------------------------------------------- |
| `data.token`    | string  | Global Token。                                      |
| `data.needInit` | boolean | 是否需要初始化工作区。当前接入流程中，用户未注册时服务端可自动完成初始化，通常返回 `false`。 |

## 4. 使用 Global Token 换取 Region Token

### 接口

* 方法：`POST`

* 路径：`/api/auth/regionToken`

* 服务：`desktop`

### 请求头

| 请求头             | 必填 | 说明                                 |
| --------------- | -- | ---------------------------------- |
| `Authorization` | 是  | 直接传 Global Token，不需要 `Bearer ` 前缀。 |

### 请求示例

```bash
curl -s -X POST 'https://<sealos-domain>/api/auth/regionToken' \
  -H 'Authorization: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.xxx'
```

### 成功响应

```json
{
  "code": 200,
  "message": "Successfully",
  "data": {
    "kubeconfig": "apiVersion: v1\nclusters:\n- cluster:\n    ...",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.region-token",
    "appToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.app-token"
  }
}
```

### 响应字段说明

| 字段                | 类型     | 说明                               |
| ----------------- | ------ | -------------------------------- |
| `data.kubeconfig` | string | 当前用户在当前工作区下可用的 kubeconfig。       |
| `data.token`      | string | Region Token，用于 `desktop` 登录态接口。 |
| `data.appToken`   | string | App Token，部分 `desktop` 接口可使用。    |

### 调用约定

* 后续如果要使用 kubeconfig 作为鉴权信息，请先执行：

```javascript
const authorization = encodeURIComponent(kubeconfig);
```

* 然后把编码后的结果放入 `Authorization` 请求头。

## 5. 获取用户 kubeconfig

### 接口

* 方法：`GET`

* 路径：`/api/auth/getKubeconfig`

* 服务：`desktop`

### 说明

返回当前 Region Token 或 App Token 对应用户在当前工作区下的 kubeconfig。服务端会自动把 kubeconfig 的 namespace 切换到当前工作区。

### 请求头

| 请求头             | 必填 | 说明                                                       |
| --------------- | -- | -------------------------------------------------------- |
| `Authorization` | 是  | Region Token 或 App Token，直接传 token 字符串，不需要 `Bearer ` 前缀。 |

### 请求示例

```bash
curl -s 'https://<sealos-domain>/api/auth/getKubeconfig' \
  -H 'Authorization: <region-token>'
```

### 成功响应

```json
{
  "code": 200,
  "message": "Successfully",
  "data": {
    "kubeconfig": "apiVersion: v1\nclusters:\n- cluster:\n    ..."
  }
}
```

### 失败响应示例

```json
{
  "code": 401,
  "message": "invalid token",
  "data": null
}
```

## 6. 轮转 kubeconfig

### 接口

* 方法：`POST`

* 路径：`/api/auth/rotateKubeconfig`

* 服务：`desktop`

### 说明

触发当前用户 kubeconfig 轮转，并返回新的 kubeconfig。服务端会轮询用户资源状态，直到拿到新的 kubeconfig 或超时。

### 请求头

| 请求头             | 必填 | 说明                                           |
| --------------- | -- | -------------------------------------------- |
| `Authorization` | 是  | Region Token，直接传 token 字符串，不需要 `Bearer ` 前缀。 |

说明：

* 该接口当前只接受 Region Token，不接受 App Token。

### 请求示例

```bash
curl -s -X POST 'https://<sealos-domain>/api/auth/rotateKubeconfig' \
  -H 'Authorization: <region-token>'
```

### 成功响应

```json
{
  "code": 200,
  "message": "Kubeconfig rotated successfully",
  "data": {
    "kubeconfig": "apiVersion: v1\nclusters:\n- cluster:\n    ..."
  }
}
```

### 失败响应示例

认证失败：

```json
{
  "code": 401,
  "message": "invalid token",
  "data": null
}
```

轮转失败：

```json
{
  "code": 500,
  "message": "Failed to rotate kubeconfig",
  "data": null
}
```

## 7. 校验 kubeconfig

### 接口

* 方法：`POST`

* 路径：`/api/auth/validateKubeconfig`

* 服务：`desktop`

### 说明

该接口通过读取 kubeconfig 当前上下文中的 namespace 来校验 kubeconfig 是否可用。

校验结果分为三类：

* 参数缺失：返回 `400`

* kubeconfig 无效但请求成功完成：返回 `200`，其中 `data.valid = false`

* 服务端内部异常：返回 `500`

### 请求头

```http
Content-Type: application/json
```

### 请求体

| 字段           | 类型     | 必填 | 说明                                  |
| ------------ | ------ | -- | ----------------------------------- |
| `kubeconfig` | string | 是  | 原始 kubeconfig 内容，放在请求体中，不需要 URL 编码。 |

### 请求示例

```bash
curl -s -X POST 'https://<sealos-domain>/api/auth/validateKubeconfig' \
  -H 'Content-Type: application/json' \
  -d '{
    "kubeconfig": "apiVersion: v1\nclusters:\n- cluster:\n    ..."
  }'
```

### 成功响应

有效：

```json
{
  "code": 200,
  "data": {
    "valid": true
  }
}
```

无效或已失效：

```json
{
  "code": 200,
  "data": {
    "valid": false
  }
}
```

### 失败响应示例

缺少参数：

```json
{
  "code": 400,
  "message": "kubeconfig is required",
  "data": null
}
```

### 请求示例

```bash
curl -s 'https://<template-domain>/api/v2alpha/templates?language=zh'
```

```json
        "type": "string",
        "default": "",
```

### 响应字段说明

| 字段             | 类型        | 说明                  |
| -------------- | --------- | ------------------- |
| `name`         | string    | 模板名称。               |
| `resourceType` | string    | 固定为 `template`。     |

```json
{
  "error": {
    "type": "internal_error",
```

## 10. 从模板目录部署应用到用户集群

### 接口

* 方法：`POST`

* 路径：`/api/v2alpha/templates/instances`

* 服务：`template`

### 说明

根据模板目录中的模板，直接向当前用户工作区部署一个实例。

对应实现位置：

* `providers/template/src/pages/api/v2alpha/templates/instances.ts`

### 鉴权

该接口要求使用 kubeconfig 鉴权。

`Authorization` 请求头需要传：

```javascript
encodeURIComponent(kubeconfig);
```

不需要 `Bearer ` 前缀。

### 请求头

```http
Content-Type: application/json
Authorization: <encodeURIComponent(kubeconfig)>
```

### 请求体

| 字段         | 类型     | 必填 | 说明                                                                                |
| ---------- | ------ | -- | --------------------------------------------------------------------------------- |
| `name`     | string | 是  | 要创建的实例名称。必须满足 Kubernetes DNS subdomain 规则：仅支持小写字母、数字、`-`，且必须以字母或数字开头和结尾，长度不超过 63。 |
| `template` | string | 是  | 模板名称，来自 `/api/v2alpha/templates` 返回的 `name` 字段。                                   |
| `args`     | object | 否  | 模板参数键值对。仅当模板参数没有默认值且被标记为必填时，才必须传入。                                                |

### 请求示例

```bash
curl -s -X POST 'https://<template-domain>/api/v2alpha/templates/instances' \
  -H 'Content-Type: application/json' \
  -H 'Authorization: <encodeURIComponent(kubeconfig)>' \
  -d '{
    "name": "my-perplexica",
    "template": "perplexica",
    "args": {
      "OPENAI_API_KEY": "sk-xxxxxxxxxxxxxxxx",
      "OPENAI_MODEL_NAME": "gpt-4o"
    }
  }'
```

### 成功响应

说明：

* 该接口成功时返回 `201 Created`。

* 响应体直接返回实例对象，不使用 `code/data` 包装。

```json
{
  "name": "my-perplexica",
  "uid": "0f987555-ca63-4d1a-8593-cae8af4a9d74",
  "resourceType": "instance",
  "displayName": "",
  "createdAt": "2026-04-02T10:22:33Z",
  "args": {
    "OPENAI_API_KEY": "sk-xxxxxxxxxxxxxxxx",
    "OPENAI_MODEL_NAME": "gpt-4o"
  },
  "resources": [
    {
      "name": "my-perplexica",
      "uid": "84219e3b-2ece-454e-b187-419ddd0841d8",
      "resourceType": "deployment",
      "quota": {
        "cpu": 1,
        "memory": 2,
        "storage": 0,
        "replicas": 1
      }
    },
    {
      "name": "my-perplexica-service",
      "uid": "216afca9-a9dd-4a20-8c98-766d4762caaf",
      "resourceType": "service"
    }
  ]
}
```

### 响应字段说明

| 字段             | 类型     | 说明                           |
| -------------- | ------ | ---------------------------- |
| `name`         | string | 创建后的实例名称。                    |
| `uid`          | string | Instance 资源的 Kubernetes UID。 |
| `resourceType` | string | 固定为 `instance`。              |
| `displayName`  | string | 当前实现中通常返回空字符串。               |
| `createdAt`    | string | Instance 资源创建时间，ISO 8601 格式。 |
| `args`         | object | 最终生效的模板参数，包含用户传入值和默认值合并后的结果。 |
| `resources`    | array  | 创建出的子资源列表，不包含 Instance 本身。   |

`resources` 中每个元素的结构如下：

| 字段             | 类型     | 说明                                                        |
| -------------- | ------ | --------------------------------------------------------- |
| `name`         | string | 子资源名称。                                                    |
| `uid`          | string | 子资源 Kubernetes UID。                                       |
| `resourceType` | string | 子资源类型，通常为小写 kind，例如 `deployment`、`statefulset`、`service`。 |
| `quota`        | object | 仅部分资源会返回，例如 `Deployment`、`StatefulSet`、`Cluster`。         |

`quota` 中可能包含以下字段：

| 字段         | 类型     | 说明          |
| ---------- | ------ | ----------- |
| `cpu`      | number | CPU 核数。     |
| `memory`   | number | 内存，单位为 GiB。 |
| `storage`  | number | 存储，单位为 GiB。 |
| `replicas` | number | 副本数。        |

### 失败响应示例

缺少 kubeconfig：

```json
{
  "error": {
    "type": "authentication_error",
    "code": "AUTHENTICATION_REQUIRED",
    "message": "Invalid or missing kubeconfig."
  }
}
```

实例名不合法：

```json
{
  "error": {
    "type": "validation_error",
    "code": "INVALID_VALUE",
    "message": "Instance name must start and end with a lowercase letter or number, and can only contain lowercase letters, numbers, and hyphens."
  }
}
```

缺少必填模板参数：

```json
{
  "error": {
    "type": "validation_error",
    "code": "INVALID_VALUE",
    "message": "Missing required parameters: OPENAI_API_KEY."
  }
}
```

实例已存在：

```json
{
  "error": {
    "type": "resource_error",
    "code": "ALREADY_EXISTS",
    "message": "Instance 'my-perplexica' already exists.",
    "details": "instances.app.sealos.io \"my-perplexica\" already exists"
  }
}
```

## 11. 获取应用商店部署实例详情

### 接口行为说明

* 服务端会尝试查询以下资源：Deployment、StatefulSet、ConfigMap、Service、Ingress、Secret、HorizontalPodAutoscaler。

* 其中部分资源不存在时会被忽略；但如果核心工作负载不存在，接口会返回错误。

* 返回的 data 为适配后的前端应用详情模型，不是某个单独 Kubernetes 资源的原始返回。

### 失败响应

```json
{"code": 500,"message": "Server error","error": "appName is empty"}
```

### 常见失败场景

* 未传 appName

* 未携带或携带了无效的 Authorization

* 指定应用不存在，或核心工作负载解析失败
